# Sesión 13: Ensamblaje del proyecto integrador

## Objetivo

Unificar el trabajo de las unidades previas en una sola aplicación consistente, alineando modelo, GUI y persistencia alrededor del caso final.

## Contenidos

- Revisión de alcance del proyecto.
- Integración de módulos ya construidos.
- Consistencia entre paquetes, nombres y flujo.
- Revisión de dependencias y recursos.

## Práctica guiada

- Consolidar el repositorio del proyecto.
- Integrar pantallas, servicios y DAO.
- Verificar el flujo principal de punta a punta.

## Reto de sesión

Detectar componentes duplicados o inconsistentes y dejar una sola versión estable.

## Entregable mínimo

Proyecto integrador ensamblado y ejecutable, aunque aún pendiente de refinamientos finales.
