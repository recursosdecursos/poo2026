# Sesión 3: Modelado del dominio, asociaciones y colecciones

## Objetivo

Representar relaciones entre objetos usando asociaciones, composición y colecciones para modelar el comportamiento del dominio.

## Preparación en IntelliJ

- Crear el paquete `app.service` o `app.model` según la organización elegida.
- Incorporar `ArrayList` para almacenar objetos relacionados.

## Contenidos

- Asociación y composición.
- Colecciones de objetos.
- Navegación entre objetos del dominio.
- Cálculo derivado, por ejemplo subtotal y total.

## Práctica guiada

- Asociar `DetalleVenta` con `Producto`.
- Componer `Venta` con una lista de detalles.
- Calcular subtotales y total general.

## Reto de sesión

Permitir agregar y quitar detalles de una venta manteniendo coherencia en el total.

## Entregable mínimo

Modelo de dominio funcional con relaciones entre objetos y operaciones básicas sobre colecciones.
